Q24. Explain about streaming stored audio/video.

Streaming Stored Audio and Video

Downloading the files of streaming stored audio/video from a web server is different compared to downloading other kinds of files. Two of the approaches for downloading these files are as follows:

1. Using a Web Server

This is a simple technique where streaming is not required. A compressed audio/video file is downloaded as a text file in the following manner:

  1. The client utilizes HTTP services to send a GET message for downloading the file.
  2. The compressed file is forwarded to the client by the web server.
  3. The client takes help of a media player to play the file.

Drawbacks:

  • Consumes more memory.
  • The compressed audio/video file is huge which is larger than the audio file.
  • Requires completely downloaded file before it can be played.

2. Using RTSP (Real-Time Streaming Protocol)

To improve the streaming process, a control protocol named Real-Time Streaming Protocol (RTSP) is utilized which controls the playing of audio/video. RTSP is an out-of-band protocol that is equivalent to the second connection which is FTP. Following steps are involved in this technique:

  1. HTTP client requests for the web server for the metafile using a GET message.
  2. The web server responds with information about the metafile.
  3. Metafile is forwarded to the media player, which sends a SET UP message to establish a connection with the media server.
  4. The media server replies.
  5. A PLAY message is sent by the media player to start playing or downloading.
  6. Downloading of audio/video file is done through a protocol that runs over UDP.
  7. TEARDOWN message is sent to the media player to terminate the connection.
  8. The media server replies.

Media player can also send other messages like, a PAUSE message that stops the downloading for sometime and can be resumed with a PLAY message later.
