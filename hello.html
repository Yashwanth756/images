Q11. Discuss the different algorithms for broadcast routing and compare them.

Broadcast, Multicast

Answer:

Broadcast Routing

The process of sending a packet to all the nodes on the network at a single instance of time is called broadcasting. The algorithms used for broadcasting are called broadcast routing.

Various methods for broadcast routing are:

  1. Distinct point-to-point routing
  2. Flooding
  3. Multi-destination routing
  4. Use of spanning tree
  5. Reverse path forwarding

1. Distinct Point-to-Point Routing

This is the simplest method for broadcasting. In this method a sender sends a distinct packet to all the nodes in the network. Thus it takes no special features of the subnet. This method is not desirable due to two reasons. First, the source must know a list of all destinations. Second, the bandwidth is wasted.

2. Flooding

Flooding algorithm send a packet on every outgoing line except the line on which it arrived. It has the same drawbacks of point-to-point algorithm. It generates too many duplicate packets and requires more bandwidth.

3. Multi-destination Routing

In multi-destination routing a packet holds a list of destinations or a bit map of desired destinations. When a router receives a packet for broadcasting, it decides the number of output lines that are needed by examining each destination. An output line is required if it contains an optimum route to at least one of the destinations. New copies of packets are created for each output line. The packet includes the list of destinations that can use the line.

4. Use of Spanning Tree

Another method of broadcasting is to use the sink tree for the router willing to broadcast a packet or to use the spanning tree.

A spanning tree is a tree that includes all the routers of the subnet but without any loops. If each router knows which of it is included in the spanning tree then it can broadcast the incoming packet on all the lines of the spanning tree except the line it arrived on. This requires minimum number of packets to be generated and the bandwidth is also efficiently utilized.

The problem with this method is that each router must know a spanning tree useful for it. With link state routing, a router has this information but with distance vector routing this information is not available.

5. Reverse Path Forwarding

In reverse path forwarding algorithm, a router that receives a packet for broadcasting checks the line the packet arrived on. If the packet arrived on the same line that is normally used for sending packets to the sender of the broadcast packet then it forwards the duplicates of packet onto all the lines except the one it arrived on. Packets are routed in reverse direction because the router assumes that the packet itself followed the best route from the sender. If packet arrived on a line other than the line preferred for sending packets to the sender then the packet is discarded assuming that it is duplicate.

All other packets are discarded as they are duplicates.

Advantages of Reverse Path Forwarding

  1. The reverse path forwarding is an efficient and easy to implement broadcast routing.
  2. The routers need not have knowledge of spanning trees.
  3. Unlike multi-destination routing, it does not have the overhead of a list of destinations or a bit map.
  4. Unlike flooding, it does not require a mechanism to stop the broadcasting process.
Q12. Explain multicast routing in detail.

Answer:

Multicast Routing Algorithm

Sending a message to multiple receivers with a single send operation is called multicasting. To achieve multicasting, multiple receivers who are interested in receiving a multicast packet form a group that receives the packets. The members of a group may join or leave a group. When a process joins a particular group, it informs about its membership to its host. The routers learn about group membership of their hosts in two ways.

  1. Host inform to their routers.
  2. Routers get this information by asking their host periodically.

The algorithm used for multicasting is called multicast routing. It works as follows,

  1. Each router builds a spanning tree that includes all other routers in a subnet. Some hosts attached to a router may belong to more than one group.
  2. When a process sends a packet to be received by a multicast group say A, then the first router in a subnet examines its spanning tree and removes all the links that don't belong to A. This is called pruning a spanning tree.
  3. Multicast packets are then routed only on the links of the appropriate spanning tree.

The simplest way of pruning the spanning tree is possible with the use of link state routing. Here each router is aware of the complete topology of a subnet. Thus, they know the group membership of their hosts. Pruning is done by moving from end of each path to the root of the spanning tree and removing all the links to routers that don't lead to hosts that are members of the group.

Another way of pruning is to use distance vector routing which uses reverse path forwarding algorithm. In this method, a router responds to a multicast message for a particular group with a prune message in the following cases,

  1. When the hosts of a router are not interested in that group and that router is not connected to other routers.
  2. When a router has no group membership.

The prune message tells the sender not to send any more multicast messages for that group to the sending router. This method recursively prunes the subnet.

Merits

  1. Multicast routing is the best way to send messages to well defined large sized groups. The size must be small compared to the size of network.
  2. It is useful in applications which require independent processes to work in collaboration with each other. For example, a group of processes that implement a distributed database.
  3. It is useful more in situations when a single process needs to send a message to all other members of a group.

Limitation

Multicast routing algorithm is not suitable for large networks because the storage requirements needed to store all the trees is considerably high. For example, a network with 'n' groups each with an average of members needs a storage for 'mn' trees, where 'm' is the number of pruned spanning trees to be stored for each group.

Applications of Multicasting

The applications of multicasting are as follows,

  1. Accessing of Distributed Databases: Usually the information in distributed databases are stored at multiple locations and accessing these locations requires the knowledge of the information storage. In such situation, whenever a user make a request, it is multicasted to all the storage location in the distributed database.
  2. News and Information Dissemination: News and the information sent by many business organizations to their customers are multicasted because a single message need to be delivered to most of the people. Example is, the delivery of any software update to all the purchasers of that particular software package.
  3. Teleconferencing: Teleconferencing involves the use of multicasting as the same information has to be simultaneously delivered to all the people who are currently attending teleconferencing. For this temporary and permanent groups are formed among the people.
  4. Distance Learning: Multicasting is also used in distance learning where the lecture delivered by a single professor can be received by a group of students residing at different places. This is useful for those students who finds difficulty in attending campus classes.
Q13. Differentiate between broadcasting and multicasting.

Answer:

Broadcasting

  • The process of sending a packet to all the destinations connected to a network is called broadcasting.
  • A network interface card accepts each incoming broadcast packet and passes it to the operating system. The CPU handles each packet. It decides whether to accept the packet or reject the packet.
  • Broadcasting is useful in applications such as distributing weather reports, stock market updates, live radio programs and TV broadcast.

Multicasting

  • The process of sending a packet to all the members of a multicast group is called multicasting. In multicasting, all those destinations interested in receiving multicast packet join a multicast group.
  • In multicasting the NIC handles each packet instead of the CPU. The CPU is used only to configure the NIC.
  • The NIC contains a list of multicast addresses that it can accept. The NIC accepts a packet if it matches any of the specified multicast addresses, the station's unicast address or the broadcast address. Otherwise, the NIC rejects the packet.
  • Multicasting is useful in applications such as distributed database systems, video conferences, distance learning and resource location.
